Communication

To implement a useful collaborative ontology development system it is not enough to provide only a shared ontology. The framework should also provide the means to build other components that support in the process of collaborative ontology development.

Furthermore communication should be reliable and ideally fault-tolerant. This was accomplished by transactional message propagation. ECF provides a special ISharedObject implementation called TransactionSharedObject for all-or-nothing, in-order messaging. Using this feature ensures that changes are either applied correctly or dropped. No partial changes can be applied which greatly reduces the risk of raising inconsistencies in case of communication errors.
